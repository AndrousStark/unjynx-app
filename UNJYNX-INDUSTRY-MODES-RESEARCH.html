01Platform Identity & Chosen Architecture

UNJYNX = Jira (Project Management) + Slack (Team Communication) + AI-Powered Multi-Channel Reminders + Industry Modes

For teams of 1-500: startups, agencies, law firms, clinics, construction crews, dev teams, marketing teams, HR departments, schools, families.

Architecture: Horizontal platform with vertical industry overlays (Salesforce Industry Cloud pattern). One codebase, 11 modes that transform vocabulary, templates, widgets, AI behavior, and workflows.

02Multi-Tenancy: Hybrid RLS Model (Chosen)

Decision: Shared DB + org_id + PostgreSQL Row-Level Security (RLS)
Validated by: Linear (shared tables, Cloud SQL, tens of TB), Slack (workspace_id on every table), Notion (shared infra with logical isolation). Optional schema-per-tenant for Enterprise compliance.

05Multi-Organization Data Model

5.1 New Tables Required


-- Organizations (multi-tenant root entity)
CREATE TABLE organizations (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name            TEXT NOT NULL,
  slug            TEXT UNIQUE NOT NULL,
  logo_url        TEXT,
  mode_id         UUID REFERENCES industry_modes(id) ON DELETE SET NULL,
  owner_id        UUID NOT NULL REFERENCES profiles(id),
  plan            TEXT NOT NULL DEFAULT 'free',   -- free | pro | team | enterprise
  settings_json   JSONB NOT NULL DEFAULT '{}',    -- timezone, branding, etc.
  is_personal     BOOLEAN NOT NULL DEFAULT false, -- true = user's personal workspace
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Organization Members (many-to-many users ↔ orgs)
CREATE TABLE organization_members (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id          UUID NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
  user_id         UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  role            TEXT NOT NULL DEFAULT 'member', -- owner | admin | member | viewer | guest
  joined_at       TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE(org_id, user_id)
);

-- Add org_id to existing tables (tasks, projects, etc.)
ALTER TABLE tasks ADD COLUMN org_id UUID REFERENCES organizations(id) ON DELETE CASCADE;
ALTER TABLE projects ADD COLUMN org_id UUID REFERENCES organizations(id) ON DELETE CASCADE;

-- Organization-scoped mode overrides (custom vocabulary per org)
CREATE TABLE org_vocabulary_overrides (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id          UUID NOT NULL REFERENCES organizations(id) ON DELETE CASCADE,
  original_term   TEXT NOT NULL,
  custom_term     TEXT NOT NULL,
  UNIQUE(org_id, original_term)
);

5.2 Migration Strategy

Backwards Compatible: Existing users get an auto-created “Personal” organization with is_personal = true. All existing tasks/projects are assigned to this personal org. No data loss, no breaking changes.

18.2 Vocabulary Translation Hook (Frontend)


// React Hook — useVocabulary()
// Resolves: org override > mode vocabulary > system default

const VocabularyContext = createContext<Record<string, string>>({});

function useVocabulary() {
  const vocab = useContext(VocabularyContext);
  return (key: string) => vocab[key] ?? key;
}

User intent → Plan steps
  → Execute step 1 (API call)
